# Architecture Decision Records (ADRs)

An ADR is a short document recording a single architectural decision: the **context**, the **decision**, and the **consequences**. We use them to capture decisions that change *after* Session 1 — so the reasoning is preserved when the original document in [`decisions/`](../decisions/index.md) is updated.

If something in `decisions/` is being revised, write an ADR rather than silently editing the original.
